В поиске поддерживаются следующие операторы:

+Предшествующий слову знак "плюс" показывает, что это слово должно присутствовать в каждом результате.
-Предшествующий слову знак "минус" означает, что это слово не должно присутствовать в результатах.
По умолчанию(если ни плюс, ни минус не указаны) данное слово расценивается как необязательное.
< >Эти два оператора используются для того, чтобы изменить вклад слова в величину релевантности, которое приписывается строке. Оператор < уменьшает этот вклад, а оператор > - увеличивает его.
( )Круглые скобки группируют слова в подвыражения.
~Предшествующий слову знак "тильда" воздействует как оператор отрицания, обуславливая негативный вклад данного слова в релевантность строки. Им отмечают нежелательные слова. Строка, содержащая такое слово, будет оценена ниже других, но не будет исключена совершенно, как в случае оператора - "минус".
*Звездочка является оператором усечения. В отличие от остальных операторов, она должна добавляться в конце слова, а не в начале.
"Фраза, заключенная в двойные кавычки, соответствует только строкам, содержащим эту фразу, написанную буквально.
@Символ "собака" вначале поискового запроса переводит поиск в режим поиска по метаданным. Доступны следующие типы метаданных:
@id- поиск по идентификатору файла в базе данных
@md5- поиск по md5-хэшу файла
@artist- поиск аудиофайлов по исполнителю
@genre- поиск аудиофайлов по жанру
@file- поиск по имени файла
@path- поиск по имени директории

Например:
  строка "@id 100500" выдаст файл имеющий идентификатор 100500
  строка "@genre Ambient" выдаст аудиофайлы в жанре Ambient
